# homeserver-vm → microvm.nix Migration

**Date:** 2026-04-22
**Status:** Approved

## Problem

`homeserver-vm` runs Vaultwarden and Syncthing in a QEMU/KVM VM orchestrated by
`scripts/vm.sh`. This requires manual `nix run '.#vm' -- create homeserver-vm` bootstrap,
nixos-anywhere for provisioning, and imperative shell for lifecycle management. Boot is slow
(full firmware + kernel init) and the workflow is friction-heavy.

## Goal

Run homeserver-vm as a **systemd service on main** via microvm.nix — declarative, sub-30s
boot, no manual orchestration. Keep `scripts/vm.sh` archived for testing impermanence on
main hardware before real hardware deployment.

## Architecture

### Overview

Five concrete changes:

1. **New flake input** — `microvm.nix` (`github:astro/microvm.nix`)
2. **New profile** — `modules/nixos/profiles/microvm-guest.nix` — replaces `vm.nix` for
   microvm guests. No disko, no UEFI bootloader, no virtio disk modules. Configures tmpfs
   root; `/persist` is a block volume (declared by the microvm module); age key is injected
   via a separate virtiofs share.
3. **`hosts/homeserver-vm/default.nix`** — swaps `vm.nix` import for `microvm-guest.nix`.
   Services, sops, and impermanence config are unchanged.
4. **`modules/nixos/microvms/homeserver-vm.nix`** — declares the microvm instance:
   cloud-hypervisor backend, volumes, virtiofs shares, TAP networking. Imported by
   `hosts/main/default.nix`.
5. **`lib/hosts.nix`** — remove `sshPort`, `diskSize`, and `deploy` from the homeserver-vm
   entry (QEMU/deploy-rs artifacts). Add `ip = "10.0.100.2"`. Keep `role`.

The NixOS smoke test (`tests/nixos/homeserver-vm-smoke.nix`) is unaffected — it uses
NixOS testing-python directly and never relied on QEMU machinery.

### Hypervisor

cloud-hypervisor — fast boot (<10s typical), good virtio-net/virtiofs support, actively
maintained. QEMU is the fallback if cloud-hypervisor has compatibility issues.

## Networking

Host-only TAP bridge on `main`:

| Host          | Interface    | IP         |
| ------------- | ------------ | ---------- |
| main          | microvm-br0  | 10.0.100.1 |
| homeserver-vm | tap (VM NIC) | 10.0.100.2 |

- Bridge declared in main's NixOS config (`networking.bridges`, `networking.interfaces`) —
  no manual `ip link` needed.
- NAT masquerading via `networking.nat` so the VM can reach the internet through main
  (needed for Syncthing peer connections).
- Tailscale and Mullvad are unaffected — neither routes to 10.0.100.0/24.
- `~/.ssh/config` entry updated: `Host homeserver-vm → 10.0.100.2` so `ssh homeserver-vm`
  still works.
- `lib/hosts.nix`: add `ip = "10.0.100.2"` to homeserver-vm — single source of truth for
  the VM's IP, referenced by the microvm module and SSH config.

## Storage & Impermanence

- **Root**: tmpfs (microvm.nix native — `fileSystems."/" = { device = "none"; fsType = "tmpfs"; }`)
- **Persist volume**: one raw ext4 image (`homeserver-vm-persist.img`, ~10 GB) stored at
  `/var/lib/microvms/homeserver-vm/persist.img` on main, mounted as `/persist` inside the
  VM via `microvm.volumes`.
- Impermanence module config in `hosts/homeserver-vm/default.nix` is unchanged — all
  service state dirs continue to bind-mount from `/persist`.

## Secrets

### Bootstrap Problem

On first boot `/persist` is empty → no SSH host key → sops cannot derive an age key →
`user_password` secret cannot be decrypted → user creation fails.

### Solution: virtiofs age key injection

**Prerequisite**: main needs a sops secrets file (`hosts/main/secrets/secrets.yaml`) and
its age key in `.sops.yaml` — create these if not already present.

1. Generate a dedicated age key pair for homeserver-vm (`age-keygen -o vm-age-key.txt`).
2. Store the private key as a sops secret on **main** (encrypted by main's age key).
3. Main decrypts it at runtime and shares it into the VM via `microvm.shares` (virtiofs)
   mounted at `/run/age-keys/` inside the VM.
4. VM's sops config:
   ```nix
   sops.age.sshKeyPaths = [];  # disable SSH-key derivation
   sops.age.keyFiles = [ "/run/age-keys/homeserver-vm.txt" ];
   ```
5. Re-encrypt `hosts/homeserver-vm/secrets/secrets.yaml` with the new age public key.
6. Update `.sops.yaml` to replace the homeserver-vm SSH host key entry with the new age
   public key.

SSH host key in `/persist` is still generated by openssh on first boot — it is just no
longer used for sops.

## scripts/vm.sh Deprecation

Add a deprecation header to `scripts/vm.sh`:

```bash
# ARCHIVED — use microvm.nix for homeserver-vm day-to-day development.
# Kept for validating impermanence/bootloader/LUKS on main hardware before
# real deployment. See docs/superpowers/specs/2026-04-22-microvm-homeserver-design.md
```

The script and its flake wiring (`apps.${system}.vm`) are kept intact. The `vm` and
`homeserver-vm` entries in `lib/hosts.nix` are updated as described — `vm` is unchanged,
`homeserver-vm` loses the QEMU-specific fields.

## Success Criteria

- homeserver-vm boots in <30 seconds after `nh os switch` on main
- Vaultwarden reachable at `https://10.0.100.2:8443` from main
- Syncthing GUI reachable at `http://10.0.100.2:8384` from main
- `ssh homeserver-vm` works
- `nix build '.#checks.x86_64-linux.homeserver-vm-smoke'` still passes
- `nix build '.#checks.x86_64-linux.invariants-homeserver-vm'` still passes
- `scripts/vm.sh` has deprecation header, `nix run '.#vm'` still works
