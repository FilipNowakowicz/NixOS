# Superpowers Design Records

This directory contains historical specs and implementation plans. Treat them
as design records and task archaeology, not as canonical documentation for the
current repo state.

Canonical current docs:

- `README.md` - overview and host inventory.
- `CLAUDE.md` - agent/operator instructions.
- `docs/architecture.md` - structural rules.
- `docs/operations.md` - deployment and validation runbook.
- `docs/security.md` - secrets, exposure, and hardening model.

Some files in `plans/` and `specs/` intentionally preserve old paths,
intermediate decisions, or pre-migration assumptions. If a design record
conflicts with live code or the canonical docs above, prefer the live code and
update the canonical docs if needed.
